NewWave Migration Tool Specification

Background :
For many years NewWave has provided the best, the most
functional and productive environment for end users on top of
MS Windows 3.1®. However the new generation of PC Client
operating systems, namely MS Windows NT 4.0® and Windows 95®, do
provide an object interface with drag and drop like NewWave,
long file names and shortcuts to network drives. Nearly all the
nice NewWave features are incorporated in the new operating
systems. Besides the NewWave features they have a lot of other
funtionallity that most end users and their administrators like.
It is a natural path to migrate from NewWave to Windows NT 4.0®
or Windows 95®.

The purpose of the tool :
The purpose of the migration tool is to extract as much of the
user’s data as possible from the NewWave environment and to make
it available to the user in a manner as close as possible to the
original NewWave environment.
Due to the complexity of the NewWave environment and lack of
direct equivalents to some aspects of NewWave in the new
operating systems, this tool will migrate the aspects of the
NewWave environment which make sense and leave some other
aspects to be manually migrated or ignored as is appropriate.

Description :
The migration is done by an export of data from NewWave and an
import of data into Windows 95® or Windows NT 4.0 ®. The export
is done by a NewWave agent task and the import is done by running
a program after upgrading to the new operating system.
The files are exported with their path and corresponding long
NewWave file names. The import creates the file structure,
copies the files and renames them to their long file names.

Features :

For objects that have multiple files attached, all files are
exported.

If two objects have the same name at the same NewWave location,
the import will add a #<number> for the second file.

Text Note documents are converted to MS Notepad documents.

NewWave Write documents are converted to MS Wordpad documents (
text only ).

Distribution lists can be converted to MS Notepad files. You should
consider very carefully if you want to migrate them, as it will
expand the migration task execution time a lot.

File containers are migrated. If the name of the container contains
a file name ( in 8.3 format ), the object is renamed to this file
name in the new environment. If not the filename will be <name>.bin .

All masters are placed in a folder called Master.

NewWave Mail objects, incomming and outgoing messages, are
treated as containers. Each message will generate a directory
during the import. The name of the directory will be the
creation date of the message, plus the name.

The tool will not move messages from NewWave/ NewWave Mail to
the mailserver. The user must manually mail the messages that
should be filed on the server to the mail server.

Object storages that are placed on a local disk ( C: and D: )
are part of the export. In the import the object storage will
create a folder, that will contain all objects that were placed
in the object storage .

The object storages that are placed on a network drive can be
copied to NewWave before the export. Else all object storages on
network drives, can be exported and moved to a new position on a
network server, in one task.

The import program operates from the command line. When you run
the program, you specify the source and the destination path. In the
.ini file for this program you specify the link between the path for
an Object Storage and the new location.

Limitations :

All documentation and programs are made in English.

The tool only supports English and Danish NewWave.

The path and name of the object cannot exceed 256 characters.

Objects with hidden parents are not migrated ( objects placed in
task folders ).

Empty objects ( data file zero bytes ) are not migrated.

The Tool will only work with NewWave version 4.10 or later.


The migration steps can be like this:

The investigation task gathers information about object types
and tool names. This information will show if you have special
things to consider in the migration.

The administrator runs the task that gathers information about
objectct storages.

The administrator uses the work sheet to plan the migration of
object storages.

Eventually the object storage task imports all or some object
storages into the NewWave environment, according to the choices
made by the administrator.

The export task is executed.

The workstation is upgraded to the new operating system.

The import program is executed.

The object storage task runs to export all of the remaining
object storages.

The import program imports the object storage data into new
network locations.

The NewWave migration tool contains:

A NewWave task to envestigate your environment.

A NewWave task to report all object storages for a user.

A NewWave task to import local or all object storages into
NewWave.

The NewWave tasks to extract data.

A program to import data.

Manual, delivered as a Ami Pro and a Word 7.0 file.

Two hours of remote support.

Everything but the support are delivered on a diskette.

Support:
All support, besides the two hours of remote support included in
the package, will be charged on a time and material basis. Bugs
that are reported will be fixed, if they are inside the scope of
the tool.

Customisation :
Your environment can contain special object types that need
special actions or your NewWave version is not Danish or English.
All request for changes to this tool will be evaluated. If your
request can be fulfilled, the change needed to the tool in order
to migrate your environment will be charged on a time and
materiel basis.
