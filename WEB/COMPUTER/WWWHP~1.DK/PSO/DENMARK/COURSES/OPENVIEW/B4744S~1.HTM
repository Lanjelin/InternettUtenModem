HP OpenView IT/Operations for systemadministratorer

    B4744S

    Formål:

    At give deltageren kendskab til de grundlæggende færdigheder, der kræves for at kunne installere HP OpenView ITO, forbedre driften, planlægge implementeringen og definere driftsopgaver i relation til HP OpenView IT/Operations.

    Teori / Øveler: 60 / 40

    Deltagere:

    Systemadministratorer og driftsansvarlige som skal kunne konfigurere og lede den daglige drift ved hjæp af HP OpenView IT/Operations.

    Forkundskaber:

    Deltageren skal have gennemgået HP OpenView grundkursus (B4739S), HP-UX 10.x systemadministration (H6296S) eller have tilsvarende viden. Derudover skal deltageren have kendskab til tcp/ip netværksprotokollen.

    Varighed:

    5 dage.

    Indhold:

    Introduktion

    • Betydningen af Management Server og Managed Node
    • Hvad kan overvåges
    • Systemer der understøtter ITO agenter
    Driftsopgaver
    • Start af OpenView IT/Operations og login som almindelig ITO bruger
    • Gennemgang af ITO vinduer
    • Identificering af nye hændelser
    • Automatiske og operatørinitierede handlinger
    • Brug af message browser
    Konfigurering af operatørens arbejdsmiljø
    • Planlægning af ansvarsområde
    • Betydningen af banks og groups
    • Konfigurering af nye applikationer
    • Integration af eksisterende OpenView applikationer
    • Opret ny operatør
    Message Management Concepts
    • Beskeder sent til management server
    • Pattern-matching expressions og message substitution
    • Betydning af matched, unmatched og suppress conditions
    • Konfigurering af message template
    Konfiguration af Message Sources
    • Konfiguration af log file template
    • Konfiguration af SNMP traps
    • Konfiguration af ITO beskeder
    • Omgruppering af meddelelser på server
    Treshold monitorering
    • Monitorinformation
    • Opsætning af tresholds på beskedfiltrering
    Actions Processing
    • Automatiske handlinger
    • Guidede handlinger
    • Ekstern event notification service
    • Integration af trouble ticket system
    Scalability aspekter
    • Koncepter omkring management server til management server kommuikation
    • Konfigurering af ITO agenter, så de kan sende beskeder til forskellige ITO-servere baseret på tid og besked attributter
    • Opsætning af ansvarsområder som tillader brug af videncentre
    • Eskalering af meddelelser mellem ITO-servere
    Hjælpeværktøjer
    • Kontrol af og status på ITO services
    • Audit kontrol
    • Rapportfaciliteter i ITO
    • Databasevedligeholdelse
    • Tilgængelige databaseværktøjer
